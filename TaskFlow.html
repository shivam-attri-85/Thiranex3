How it works

  • The list is rendered from a single state array, then re-painted after each interaction.
  • Button clicks, checkbox changes, and inline edit saves all flow through delegated listeners on the task container.
  • Filters only change the view, so the same underlying data remains intact across All, Active, and Completed modes.
  • Data survives refreshes because the current state is serialized to localStorage after every mutation.
